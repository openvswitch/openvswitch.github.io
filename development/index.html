Open vSwitch is developed as an open source project primarily under the Apache 2 license.

Using Git


If you intend to do a substantial amount of development, or to contribute patches upstream, we recommend installing Git.  Once you have Git installed, you may clone the Git repository for Open vSwitch with:

git clone https://github.com/openvswitch/ovs.git

Code Snapshots

If you do not want to use Git, you can download a snapshot in .zip format. Start from the repository webpage at github, then click on the "Download ZIP" button on the right side of the page.

How to Contribute


Contributions should be based on the latest development version of Open vSwitch.  More information about Open vSwitch development is right in the source tree or on this site:

  • Coding style
  • Guidelines for reporting bugs
  • How to submit patches
  • Committer Responsibilities
  • Committer Grant/Revocation

Before beginning work on a contribution, please review the Developer's Certificate of Origin in How to submit patches. All submitted patches are required to have a "Signed-off-by:" record in the commit message recording the developer's agreement with this certificate of origin.

Online Resources

  • Repository webpage at github

Questions about Open vSwitch development are welcome on the mailing list, at dev@openvswitch.org.
